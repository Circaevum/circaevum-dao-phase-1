# Quarterly Versioning System Example

## Version Format: `v25Q3.YANG.YIN.TAIJI`

The versioning system resets each quarter and reflects the current progress state:

### **Version Components**
- **25Q3** = Quarter and Year (2025 Q3)
- **YANG** = YANG New progress (sum of positions 1-17)
- **YIN** = YIN New progress (sum of positions 38-54)
- **TAIJI** = TAIJI Center progress (position 37)

### Key Versioning Characteristics

#### **1. Quarterly Reset**
- Each quarter starts at `v25QX.0.0.0`
- Version numbers reflect current quarter progress only
- Clean slate for new development focus

#### **2. Progress Reflection**
- **YANG**: Shows frontend development progress
- **YIN**: Shows backend development progress
- **TAIJI**: Shows coordination/integration progress

#### **3. Time-Based Progression**
- Version numbers increase as development progresses
- Higher numbers indicate more completed work
- Matches the dot progress code accumulation

#### **4. Development Focus**
- **Q1**: Foundation (lower numbers, focused work)
- **Q2**: Features (balanced growth)
- **Q3**: Integration (coordinated development)
- **Q4**: Enterprise (scaled development)

### Version vs Dot Progress Code

#### **Version**: `v25Q3.3000.27000.7`
- **Time-based**: Resets each quarter
- **Progress-focused**: Shows current quarter achievements
- **Simplified**: Easy to read and understand

#### **Dot Progress Code**: `25Q3 2-003000-002000 7 2-027000-003000`
- **Cumulative**: Shows total progress across all quarters
- **Detailed**: Includes all dot positions and sections
- **Comprehensive**: Complete state representation

This versioning system provides both a clean quarterly reset and a clear reflection of development progress, making it easy to track both current quarter achievements and overall project evolution!
